## Java 9: Optional.ifPResentOrElse() 
Xử lý gọn 2 trường hợp có giá trị và giá trị rỗng
```java
Optional<User> user = repository.findById(id);

// one fluent call - both branches handled
user.ifPresentOrElse(
	
	u -> logger.info("Welcome back, " + u.getName()),
	
	() -> logger.warn("User not found: " + id)
);
```
## Java 9: Optional.or()
Cung cấp Optional thay thế khi có giá trị rỗng
![[Pasted image 20260319093402.png]]
Đảm bảo tính **fluent API** của **Optional**
VD: tìm kiếm config
- Kiểm tra cache --> không có sang DB
- Kiểm tra DB --> ko có vào local storage configs

```java
List<Optional<String>> optionals = Arrays.asList(
	 Optional.of("A"),
	 Optional.empty(),
	 Optional.of("B") 
 ); 
List<String> values = optionals.stream() 
.flatMap(Optional::stream) // Java 9+ 
.collect(Collectors.toList()); 
// Output: ["A", "B"] (bỏ empty)
```
## Java 9: Collectors.flatMapping()
Thử thu thập danh sách lồng nhau trong groupBy().
Kết quả là Map<String, List<List< String >>> xấu xí và phải dùng thêm bước flatten thủ công
Sử dụng **Collectors.flatMapping()** để loại bỏ hoàn toàn vấn đề
- Flatten trực tiếp trong khi thu thập
- Pipeline gọn, dễ kết hợp mà không cần vòng lặp cầu kì
